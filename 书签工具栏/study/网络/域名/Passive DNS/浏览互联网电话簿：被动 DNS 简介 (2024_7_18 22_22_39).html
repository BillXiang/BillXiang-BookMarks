Navigating the Internet’s Phone Book: An Introduction to Passive DNS

浏览互联网电话簿：被动 DNS 简介

Think you already understand Passive DNS? Think again. If you investigate cyber crimes, you should be actively paying attention to DNS.

认为您已经了解被动 DNS？ 再想一想。 如果您调查网络犯罪，您应该积极关注 DNS。

“Passive DNS” or “Passive DNS replication” is a technique invented by Florian Weimer in 2004 to opportunistically reconstruct a partial view of the data available in the global Domain Name System into a central database where it can be indexed and queried. Passive DNS allows threat hunters to connect individual indicators of compromise (IOC) with nearly every active domain and IP address on the Internet. Learn how Passive DNS data can help you map cyber activity to attacker infrastructure.

“被动 DNS”或“被动 DNS 复制”是 Florian Weimer 于 2004 年发明的一种技术，用于将全球域名系统中可用数据的部分视图机会性地重建到中央数据库中，以便对其进行索引和查询。 被动 DNS 允许威胁追踪者将各个入侵指标 (IOC) 与 Internet 上的几乎每个活动域和 IP 地址连接起来。 了解被动 DNS 数据如何帮助您将网络活动映射到攻击者基础设施。

  • - How does Passive DNS work?
  • - How does DNS tie into investigating criminal activity on the Internet?
  • - How can Passive DNS be used in cyber investigations?

What Is Passive DNS?

什么是被动 DNS？

Regular DNS: The “regular” Domain Name System (“DNS”) is often referred to as the Internet’s “phone book.” It resolves a symbolic domain name (such as Farsight Security, cyber security intelligence solutions) to the IP address that computers and networks actually need (such as 104.244.14.108). The DNS usually works so well that few people think twice about it or bother to learn much about it. “It just works.”

常规 DNS：“常规”域名系统 (“DNS”) 通常被称为 Internet 的“电话簿”。 它将一个符号域名（如 Farsight Security、网络安全情报解决方案）解析为计算机和网络实际需要的 IP 地址（如 104.244.14.108）。 DNS 通常工作得很好，以至于很少有人会三思而后行或费心去了解它。 “它只是有效。”

DNS and Investigating Criminal Activity on the Internet: If you investigate cybercrimes, however, you really should be paying attention to DNS.

DNS 和调查 Internet 上的犯罪活动：但是，如果您调查网络犯罪，您确实应该关注 DNS。

Why, you ask? Well, just to mention two reasons:

你为什么问？ 好吧，简单说两个原因：

• Pretty much anything that happens on the Internet -- good or bad --- starts with a DNS lookup. This makes DNS a terrific way to figure out what sort of sites are getting visited. It is often easy to spot phishing web sites, sites being used to sell illegal drugs, or sites selling fake documents, for example.

• 互联网上发生的几乎任何事情——无论好坏——都始于DNS 查找。 这使得 DNS 成为一种确定正在访问的站点类型的极好方法。 例如，通常很容易发现网络钓鱼网站、用于销售非法毒品的网站或销售伪造文件的网站。

• Multiple sites will often share a common web server, a common mail server, or common name servers. That means that if we can find one bad site, we can often use that initial “lead” to find additional bad sites.

• 多个站点通常会共享一个公共网络服务器、一个公共邮件服务器或公共名称服务器。 这意味着，如果我们能找到一个不良站点，我们通常可以使用最初的“潜在客户”来查找其他不良站点。

Unfortunately, many of the DNS queries an investigator would LIKE to make aren’t queries that the “regular DNS” knows how to answer.

不幸的是，调查人员希望进行的许多 DNS 查询并不是“常规 DNS”知道如何回答的查询。

For example, you can’t ask the “regular DNS” to see all the domain names that have IP addresses from a particular network range -- the regular DNS simply doesn’t know how to answer that sort of question.

例如，您不能要求“常规 DNS”查看具有特定网络范围内 IP 地址的所有域名——常规 DNS 根本不知道如何回答此类问题。

Nor could you ask to see a listing of all the fully qualified domain names (“hostnames”) that exist under a particular “2nd level domain” (such as “oregon.gov”).

您也不能要求查看特定“二级域”（例如“oregon.gov”）下存在的所有完全限定域名（“主机名”）的列表。

Fortunately, if you’re using passive DNS, you CAN answer those sorts of questions.

幸运的是，如果您使用的是被动 DNS，您可以回答这类问题。

How Passive DNS Works

Passive DNS is based on passively-observed DNS queries and responses. For example, perhaps a home broadband user makes a query for www.popeyes.com, a popular (and completely legitimate) American restaurant that serves exceptionally-tasty spicy fried chicken.

被动 DNS 基于被动观察的 DNS 查询和响应。 例如，家庭宽带用户可能会查询 Popeyes: Online Offers and Ordering，这是一家受欢迎的（且完全合法的）美国餐厅，供应特别美味的辣炸鸡。

Depending on where that query gets made, a passive DNS sensor may happen to see that DNS query, and see that the hostname resolves to an IP address, such as 13.52.58.70.

根据进行查询的位置，被动 DNS 传感器可能会碰巧看到该 DNS 查询，并看到主机名解析为 IP 地址，例如 13.52.58.70。

The passive DNS service associated with that sensor will record that information in a passive DNS database (such as Farsight’s DNSDB® ).

与该传感器关联的被动 DNS 服务将在被动 DNS 数据库（例如 Farsight 的 DNSDB®）中记录该信息。

Once that information has been added to DNSDB, DNSDB users can search DNSDB for that name, thereby learning what IP address or addresses the name has resolved to.

一旦将该信息添加到 DNSDB，DNSDB 用户就可以在 DNSDB 中搜索该名称，从而了解该名称已解析为哪些 IP 地址。

Users can also query for the IP address and learn the domain name that pointed at that IP.

用户还可以查询IP地址并了解指向该IP的域名。

For example, looking at the fully qualified domain name with the DNSDB command line tool dnsdbq, 1 we see:

例如，使用DNSDB命令行工具dnsdbq，1查看完全限定域名，我们看到：

dnsdbq -r http://www.popeyes.com -S -k last | more
;; record times: 2019-11-05 15:04:13 .. 2020-04-06 23:41:49
;; count: 179249; bailiwick: popeyes.com.
www.popeyes.com. CNAME rbi.netlifyglobalcdn.com.
;; record times: 2019-08-07 16:07:03 .. 2019-11-05 15:04:00
;; count: 96401; bailiwick: popeyes.com.
www.popeyes.com. CNAME popeyes.com.
;; record times: 2017-03-31 02:33:47 .. 2019-08-07 15:59:50
;; count: 369691; bailiwick: popeyes.com.
www.popeyes.com. CNAME popeyeswcag.wpengine.com.
[ etc ]

In recent days (e.g., from 2019-11-05 15:04:13 to when this article was written) Popeyes: Online Offers and Ordering is a CNAME (or nickname) record, see the grey highlighted result.

最近几天（例如，从2019-11-05 15:04:13到写这篇文章的时候）Popeyes：Online Offers and Ordering是一个CNAME（或昵称）记录，见灰色突出显示的结果。

That CNAME points at another domain name, note the burnt orange highlighted name (rbi.netlifyglobalcdn.com).

该 CNAME 指向另一个域名，注意烧焦的橙色突出显示的名称 (rbi.netlifyglobalcdn.com)。

Note that DNSDB reported the IP address we’d found in the “real DNS”, 13.52.58.70, but our sensors also saw other IP addresses during roughly the same time, too. In this case, because Popeyes® is a hugely popular fast food franchise, the content delivery network they’re using resolves the site’s nickname (rbi.netlifyglobalcdn. com) to multiple IP addresses during the same time period (the CDN might send queries from the West Coast of the US to one server, queries from the East Coast to another server, and queries from the Deep South to a third server). This means that we can now easily see details about this domain that we might otherwise have been hard pressed to discover.

请注意，DNSDB 报告了我们在“真实 DNS”中找到的 IP 地址 13.52.58.70，但我们的传感器也大致在同一时间看到了其他 IP 地址。 在这种情况下，因为 Popeyes® 是一个非常受欢迎的快餐连锁店，他们使用的内容交付网络在同一时间段内将站点的昵称 (rbi.netlifyglobalcdn.com) 解析为多个 IP 地址（CDN 可能会发送来自 美国西海岸到一台服务器，从东海岸查询到另一台服务器，从深南部查询到第三台服务器）。 这意味着我们现在可以轻松查看有关此域的详细信息，否则我们可能很难发现这些详细信息。
