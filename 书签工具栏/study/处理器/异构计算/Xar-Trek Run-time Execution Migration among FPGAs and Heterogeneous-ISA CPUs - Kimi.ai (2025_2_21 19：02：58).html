      Xar-Trek Run-time

        总结一下 https://www.ssrg.ece.vt.edu/papers/middleware21.pdf
        本文介绍了一个名为 Xar-Trek 的框架，旨在实现应用程序在运行时在 FPGAs 和异构 ISA（指令集架构）CPU 之间动态迁移执行，以提高数据中心服务器的性能和资源利用率。

        背景知识

        随着数据中心对性能、能效和小型化的需求增加，服务器架构越来越多样化，包括 x86 主机 CPU、ARM 或 RISC-V CPU（常用于 NIC 和 SSD 中），以及 FPGAs。尽管已有研究展示了在异构 ISA CPU 之间动态迁移应用程序执行可以显著提高性能和能效，但 FPGAs 在这一领域一直被忽视。传统 FPGA 加速模型是静态的，一旦将应用程序函数映射到 FPGA，其执行就固定在 FPGA 上，无法动态迁移。

        研究方法

        Xar-Trek 通过编译器和运行时软件框架克服了这一限制。它能够为多种 CPU ISA 编译应用程序，并为特定应用程序函数生成 FPGA 加速版本，允许在运行时在异构 ISA CPU 和 FPGA 之间动态迁移执行。

        编译器框架

        Xar-Trek 的编译器框架包括以下步骤：
        1. Profiling（分析）：手动确定可在三种目标架构（x86、ARM 和 FPGA）上执行的函数。
        2. Instrumentation（代码注入）：为每个函数插入调度器客户端调用，用于与调度器服务器通信。
        3. Multi-ISA Binary Generation（多 ISA 二进制生成）：使用 Popcorn Linux 编译器生成可在 x86-64 和 ARM64 CPU 上运行的二进制文件。
        4. Xilinx Object Generation（Xilinx 对象生成）：将选定函数映射到 FPGA，生成 Xilinx 对象（XO）文件。
        5. XCLBIN Partitioning（XCLBIN 分区）：根据 FPGA 资源和硬件平台的可用性，将函数分配到一个或多个 XCLBIN 文件中。
        6. XCLBIN Generation（XCLBIN 生成）：将函数和硬件平台集成到一个 XCLBIN 文件中，并将其下载到 FPGA。
        7. Threshold Estimation（阈值估计）：通过测量应用程序在不同迁移场景下的执行时间，确定触发执行迁移的 CPU 负载阈值。

        运行时系统

        Xar-Trek 的运行时系统包括一个用户空间调度器，它动态决定每个应用程序函数的执行位置。调度器基于客户端/服务器模型，客户端与服务器通过套接字和信号通信。调度器服务器根据调度策略决定函数的执行目标。

        实验

        实验使用了 Intel Xeon x86 CPU、Cavium ThunderX ARM CPU 和 Xilinx Alveo U50 FPGA 卡。实验选择了 AI/视觉和 HPC 领域的代表性工作负载，包括人脸检测、数字识别和 NAS 并行基准测试中的 CG-A 应用。

        性能评估

        实验结果表明，Xar-Trek 在不同负载条件下均能有效提高性能：
        • 低负载：当进程数少于 x86 核心数时，Xar-Trek 的性能与仅在 x86 上运行的 Vanilla Linux 相当。
        • 中等负载：当进程数超过 x86 核心数但少于总核心数时，Xar-Trek 的性能提升显著，最大提升 88%。
        • 高负载：当进程数超过总核心数时，Xar-Trek 的性能提升在 19% 到 31% 之间。

        吞吐量评估

        Xar-Trek 在处理多图像人脸检测任务时，当 CPU 负载超过 25 个进程时，其吞吐量比仅在 FPGA 上运行的 Vanilla Linux 高出约 4 倍。

        动态工作负载评估

        在模拟数据中心动态工作负载的实验中，Xar-Trek 在周期性负载变化下表现出色，平均执行时间比 Vanilla Linux/x86 和 Vanilla Linux/FPGA 分别快 18% 和 32%。

        关键结论

        Xar-Trek 通过动态迁移应用程序函数的执行，能够在运行时根据工作负载条件在异构 ISA CPU 和 FPGA 之间灵活分配计算任务，从而显著提高数据中心服务器的性能和资源利用率。该框架的调度策略能够根据实时负载和预估阈值做出合理的迁移决策，即使在负载快速变化的情况下也能保持高效。

        限制

        尽管 Xar-Trek 取得了显著的性能提升，但它也存在一些限制：
        1. 迁移粒度：目前只能在函数级别进行迁移，无法在更细粒度的代码段或指令级别迁移。
        2. 语言支持：目前仅支持 C 语言，扩展到 C++ 等语言需要进一步工程支持。
        3. 功耗优化：当前框架主要关注性能优化，未来可以扩展到功耗优化，例如使用低功耗 ARM 芯片。
        4. FPGA 资源共享：目前不支持多个应用程序同时共享 FPGA 资源，未来可以探索在 FPGA 上实现多租户支持。

        未来工作

        未来的研究方向包括：
        1. 克服当前的限制，如支持更多语言和更细粒度的迁移。
        2. 在 FPGA 上实现多租户支持，以进一步提高资源利用率。
        3. 开发针对 FPGA 的安全机制，以防止侧信道攻击。
        4. 优化调度策略，以更好地平衡性能和功耗。
