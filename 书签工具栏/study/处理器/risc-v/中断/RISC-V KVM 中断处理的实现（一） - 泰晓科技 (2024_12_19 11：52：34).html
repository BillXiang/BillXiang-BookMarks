      RISC-V KVM 中断处理的实现（一）

      1 前言

      本文就 Spike 和 QEMU 中与 RISC-V Trap 处理相关的实现细节进行了较为深入的挖掘，包括 PLIC 和 CLINT 等在模拟器中的实现。从模拟器的角度来看，当前它们对于虚拟化扩展的支持，是在其内部中断处理机制的基础上，添加了虚拟化所需的 CSR 以及对应的 Trap 判断与触发逻辑。

      3 Spike 实现分析

      3.1 Trap 统一编码与命名习惯

      RISC-V Trap 包含中断和异常，其编码在 自动生成 的编码文件中分别以如下名称开头：

      • IRQ：Interrupt ReQuest，用于表示中断编码
      • CAUSE：cause 是 RISC-V 中用于保存 trap 的具体编码的 CSR，包括 mcause, scause, vscause 等，此处专门用于表示异常编码

      3.2 Trap 处理函数调用

      Spike 定义了 sim_t 类作为最外层的模拟管理器，其包含了一个 processor vector，Spike 模拟 CPU 的执行即是 sim 的各个 processor 调用 step 函数执行 fetch, decode, execute 等操作。trap 处理操作即在 step 函数中实现。

      3.2.1.2 判断当前是否需要进行 Interrupt 处理

      当且仅当 mie 和 mip 两个 CSR 的值，进行逻辑与不为 0 时，trap 才会被处理：

      1. void take_pending_interrupt() { take_interrupt(state.mip->read() & state.mie->read()); }
      3.2.1.3 确定 Interrupt 的内容

      void take_interrupt(reg_t mask);：如果 mask 为 0 则不执行 trap 处理，否则将判断 trap 具体类型，抛出一个 trap（trap_t）

      1. void processor_t::take_interrupt(reg_t pending_interrupts)
      2. {
      3.   // 不执行中断处理
      4.   // Do nothing if no pending interrupts
      5.   if (!pending_interrupts) {
      6.     return;
      7.   }
      9.   // 依照 M, HS, VS 的顺序确定具体要处理哪一个特权级的 trap
      10.   // M-ints have higher priority over HS-ints and VS-ints
      11.   const reg_t mie = get_field(state.mstatus->read(), MSTATUS_MIE);
      12.   const reg_t m_enabled = state.prv < PRV_M || (state.prv == PRV_M && mie);
      13.   reg_t enabled_interrupts = pending_interrupts & ~state.mideleg->read() & -m_enabled;
      14.   if (enabled_interrupts == 0) {
      15.     // HS-ints have higher priority over VS-ints
      16.     const reg_t deleg_to_hs = state.mideleg->read() & ~state.hideleg->read();
      17.     const reg_t sie = get_field(state.sstatus->read(), MSTATUS_SIE);
      18.     const reg_t hs_enabled = state.v || state.prv < PRV_S || (state.prv == PRV_S && sie);
      19.     enabled_interrupts = pending_interrupts & deleg_to_hs & -hs_enabled;
      20.     if (state.v && enabled_interrupts == 0) {
      21.       // VS-ints have least priority and can only be taken with virt enabled
      22.       const reg_t deleg_to_vs = state.hideleg->read();
      23.       const reg_t vs_enabled = state.prv < PRV_S || (state.prv == PRV_S && sie);
      24.       enabled_interrupts = pending_interrupts & deleg_to_vs & -vs_enabled;
      25.     }
      26.   }
      28.   // 按照 MEI, MSI, MTI; SEI, SSI, STI (HS > VS) 的优先级确定 mcause/scause 最高位之外的内容
      29.   if (!state.debug_mode && enabled_interrupts) {
      30.     // nonstandard interrupts have highest priority
      31.     if (enabled_interrupts >> (IRQ_M_EXT + 1))
      32.       enabled_interrupts = enabled_interrupts >> (IRQ_M_EXT + 1) << (IRQ_M_EXT + 1);
      33.     // standard interrupt priority is MEI, MSI, MTI, SEI, SSI, STI
      34.     else if (enabled_interrupts & MIP_MEIP)
      35.       enabled_interrupts = MIP_MEIP;
      36.     else if (enabled_interrupts & MIP_MSIP)
      37.       enabled_interrupts = MIP_MSIP;
      38.     else if (enabled_interrupts & MIP_MTIP)
      39.       enabled_interrupts = MIP_MTIP;
      40.     else if (enabled_interrupts & MIP_SEIP)
      41.       enabled_interrupts = MIP_SEIP;
      42.     else if (enabled_interrupts & MIP_SSIP)
      43.       enabled_interrupts = MIP_SSIP;
      44.     else if (enabled_interrupts & MIP_STIP)
      45.       enabled_interrupts = MIP_STIP;
      46.     else if (enabled_interrupts & MIP_LCOFIP)
      47.       enabled_interrupts = MIP_LCOFIP;
      48.     else if (enabled_interrupts & MIP_VSEIP)
      49.       enabled_interrupts = MIP_VSEIP;
      50.     else if (enabled_interrupts & MIP_VSSIP)
      51.       enabled_interrupts = MIP_VSSIP;
      52.     else if (enabled_interrupts & MIP_VSTIP)
      53.       enabled_interrupts = MIP_VSTIP;
      54.     else
      55.       abort();
      57.     // 抛出异常编码（最高位为 1）
      58.     throw trap_t(((reg_t)1 << (isa->get_max_xlen() - 1)) | ctz(enabled_interrupts));
      59.   }
      60. }

      由图可知，最终的 trap 处理是通过调用 riscv/process.cc 的 take_trap 函数实现的，其定义及功能分析参见 此文 的模拟器实现中的 Spike 小节。简单来说，它完成了如下任务：

      • 确定该 trap 的类型（异常或者中断）以及它将在哪个特权级被处理（默认 M，但可以通过 medeleg/mideleg 和 hedeleg/hideleg 委托给 HS 或 VS）
      • 写入 CSR（cause, epc, tval）以记录 trap 内容，保存当前执行环境用于处理完成之后的恢复
      • 修改 status 寄存器及当前特权级，进入 trap 处理程序运行环境

      3.3 CLINT

      CLINT（Core-Local INTerrupt）是 RISC-V 核间局部中断控制器，用于管理软件中断和计时器中断的注入和解除。

      在 Spike 中，CLINT 是 bus 上的设备之一 device，CPU 对 CLINT 和 mems 等的访问通过 mmio.store/load 来实现。所有的 mmio 访存均是通过调用 bus.load/store 完成的，例如 mmu_t 和 sim_t 的 load, store 操作：

      （下载由 Mermaid 生成的 PNG 图片）

      而 CPU 通过 bus 实现的访存，是先通过地址所在的范围确定对应的具体设备，进而调用该设备的访存函数来完成的：

      1. bool bus_t::load(reg_t addr, size_t len, uint8_t* bytes)
      2. {
      3.   // Find the device with the base address closest to but
      4.   // less than addr (price-is-right search)
      5.   auto it = devices.upper_bound(addr);
      6.   if (devices.empty() || it == devices.begin()) {
      7.     // Either the bus is empty, or there weren't
      8.     // any items with a base address <= addr
      9.     return false;
      10.   }
      11.   // Found at least one item with base address <= addr
      12.   // The iterator points to the device after this, so
      13.   // go back by one item.
      14.   it--;
      15.   return it->second->load(addr - it->first, len, bytes);
      16. }
      18. bool bus_t::store(reg_t addr, size_t len, const uint8_t* bytes)
      19. {
      20.   auto it = devices.upper_bound(addr);
      21.   if (devices.empty() || it == devices.begin()) {
      22.     return false;
      23.   }
      24.   it--;
      25.   return it->second->store(addr - it->first, len, bytes);
      26. }

      对于 CLINT 来说，其所担负的软件中断和计时器中断的职能，就是通过 clint_t 的 load/store 方法配合对应 CSR 的访问来实现的，参考 此文：

      1. // riscv/clint.cc: line 48
      2. bool clint_t::store(reg_t addr, size_t len, const uint8_t* bytes)
      3. {
      4.   // 若地址在 MSIP 内，表示当前中断为软件中断，将 byte 内的内容写入 addr 作为偏移量所指示的位置，即写入软件中断
      5.   if (addr >= MSIP_BASE && addr + len <= MSIP_BASE + procs.size()*sizeof(msip_t)) {
      6.     std::vector<msip_t> msip(procs.size());
      7.     std::vector<msip_t> mask(procs.size(), 0);
      8.     memcpy((uint8_t*)&msip[0] + addr - MSIP_BASE, bytes, len);
      9.     memset((uint8_t*)&mask[0] + addr - MSIP_BASE, 0xff, len);
      10.     for (size_t i = 0; i < procs.size(); ++i) {
      11.       if (!(mask[i] & 0xFF)) continue;
      12.       procs[i]->state.mip->backdoor_write_with_mask(MIP_MSIP, 0);
      13.       if (!!(msip[i] & 1))
      14.         procs[i]->state.mip->backdoor_write_with_mask(MIP_MSIP, MIP_MSIP);
      15.     }
      16.   } else if (addr >= MTIMECMP_BASE && addr + len <= MTIMECMP_BASE + procs.size()*sizeof(mtimecmp_t)) {
      17.     // 设置计时器中断：向 mtimecmp 寄存器写入特定值，待到 mtime 达到该值时产生一个中断
      18.     memcpy((uint8_t*)&mtimecmp[0] + addr - MTIMECMP_BASE, bytes, len);
      19.   } else if (addr >= MTIME_BASE && addr + len <= MTIME_BASE + sizeof(mtime_t)) {
      20.     // 设置 mtime 的值
      21.     memcpy((uint8_t*)&mtime + addr - MTIME_BASE, bytes, len);
      22.   } else {
      23.     return false;
      24.   }
      25.   increment(0);
      26.   return true;
      27. }

      3.4 PLIC/Interrupt Controller/NS16550

      PLIC（Platform-Level Interrupt Controller）是 RISC-V 外部中断控制器，而 NS16550 则是一个 UART（Universal Asynchronous Receiver/Transmitter）软核，用于 CPU 和外设之间的通信。CLINT，PLIC 以及 NS16550 都要挂载在设备树（fdt, flatten device tree）上。NS16550 在创建时会以 PLIC 指针为中断控制器，用来向 CPU 发送外部中断。

      与 CLINT 类似，PLIC 等设备在创建之后也都几乎没有用到除计时之外的功能。

      4.2 Trap 处理

      QEMU 内部通过 DEFINE_TYPES 注册 RISC-V CPU 的相关信息，其中就包括中断处理相关的操作。内部中断（Software，Timer）通过将中断处理函数 riscv_cpu_do_interrupt 注册到 tcg_ops 里面来实现，外部中断则通过 CPU 初始化函数中的 GPIO 设备配置来完成。
