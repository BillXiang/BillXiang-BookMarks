J扩展[1]的设计目标是为了需要动态翻译（dynamic translation）的语言准备的扩展指令集，如Java、JS等。

该扩展目前还在草稿阶段，最近一个月才提出一个提案：pointer-masking-proposal。该提案主要是为了内存安全机制Memory Tag和用户态下沙盒机制设计的。

Memory Tag[2]

内存安全问题主要有两大类：

●Temporal: heap-use-after-free , stack-use-after-return , stack-use-after-scope ●Spatial: heap-buffer-overflow , stack-buffer-overflow , global-buffer-overflow ,container-overflow

AddressSanitizer[3][4]是谷歌提出的一种检测内存的工具，他提供多种方式检测内存问题如：Readzone、Quarantine 。

Quarantine是在内存分配时多分配一部分内存称之为shadow memory,该块内存用来标记被申请的内存是否有效，可以用来检测Temporal问题。

Readzone用来检测Spatial（内存溢出），原理是在内存申请或分配时，将被分配的内存后面插入一块Readzone区域，这块区域被标记为Quarantine不可操作的区域，一旦内存访问到Readzone区域，代表内存发生溢出。

但也有几个问题：

有时内存访问如果越过在内存中的Redzone，那内存溢出就不会被ASAN检测到。Quarantine 需要编译器插入一些指令进行检查，这会导致代码量增加、性能下降：

●1.5x-3x in CPU, from instrumenting all loads and stores and from heavier malloc.
●1.5x-3x in RAM, from redzones, quarantine, shadow, and internal bookkeeping.
●1.5x-3x in Code Size, from instrumenting all loads and stores.

还有其他限制：

●Does not currently instrument assembly code
●Does not currently detect bugs in pre-compiled binaries
●Does not currently detect when the kernel accesses invalid user-space memory

为了改善以上限制，谷歌提出了Memory Tag。

MT主要利用了用户态下地址只需要低48bit，高24bit全为0的特点. 在地址48:62中某些bit中插入一些tag作为key,在内存申请时额外申请一块内存插入Lock。读写内存的时候检查这个Tag是否匹配，若不匹配则引发一个exception.

而这些TAG是再申请内存时随机生成的可以由软件随机生成，也可以由硬件提供支持[5][6]。

MT详细规则如下：

  • 每TG（tagging granularity）字节的对齐到TG的内存都会有一个tag,该tag占据 TS (tag size)bits.这些TG字节的内存称之为granule。
  • 在指针中高半部分的TSbits存储一个tag。
  • malloc函数选择一个tag与申请的memory chunk关联，并将返回的指针中设置tag.
  • 每次LD和ST都会读取memory tag并将其与指针tag进行比较，如果标签不匹配，则会引发异常。

Memory Tag由Arm实现的方案为AArch64 HWASAN[7][8]。

该方案主要利用Armv8的top-byte-ignore（TBI）特性(如果没有此特性，MT将会对TLB带来造成压力)，将TG设为16，TS设为8.

由于加入MT造成程序体积变大，安卓APP增大范围在0%~6%，Linux内核4%。Linux栈帧增加9%[2]。

此外ArmV8.5的MTE扩展，增加了几条关于tag的指令，可以进一步减少由于加入MT而增加的指令数量。

Pinter Masking

RISCV-J扩展中的Pinter Masking (PM)特性是Memory Tag实现的基础。除此之外PM还被用来做沙盒机制。

主要思路是利用两个寄存器 xPMMASK、xPMBASE来解析指针：

actual_address = (requested_address & ~xPMMASK) | xPMBASE

若xPMBASE为0x00，就可以用xPMMASK寄存器实现类似armv8.5中top-byte-ignore的feature.

若xPMBASE为不0x00，则用户程序在地址空间上隔离起来：

形成一种沙盒的机制，将不同用户程序进行隔离。比如JS的V8引擎中Isolate创建时可以分配一对xPMMASK、xPMBASE，彻底隔离Isolate。
