RISC-V 虚拟化差距总结
该文档（“RISC-V Virtualization Gaps”）总结了 RISC-V 在虚拟化支持方面与主流架构（如 x86_64、ARM64）相比存在的主要差距，并分类为：

1. SBI（超级二进制接口）或半虚拟化支持

  • PV-spinlocks（半虚拟化自旋锁）
    • 问题：避免虚拟机因锁持有者被抢占而空转。
    • 状态：进行中（PoC 已提交）。
  • Timescaling（定时器频率动态调整）
    • 问题：运行时向虚拟机报告定时器频率变化。
    • 状态：提案已提出（SBITSCL 扩展），但近期无进展。
  • Async Page Faults（异步页错误）
    • 问题：半虚拟化方式通知虚拟机被换出的 G-stage 页。
    • 依赖：需要 SBI 的 Supervisor Software Events（SSE）。
    • 状态：未启动。
  • 可扩展机密计算架构（CoVE）
    • 问题：支持跨 Hypervisor 的机密计算。
    • 状态：规范草案已完成，高优先级。

2. ISA（指令集架构）优化

  • 加速 G-stage 脏页跟踪
    • 问题：硬件支持更快的脏页标记。
    • 状态：未启动，低优先级。
  • PMU（性能监控单元）增强
    • 问题：主机和虚拟机均需支持 PMU 快照。
    • 状态：ISA 部分进行中，SBI 部分未启动，高优先级。
  • MMU-less 虚拟化（SPMP 扩展）
    • 问题：通过 SPMP（Supervisor Physical Memory Protection）支持无 MMU 的虚拟化。
    • 状态：进行中。

3. 非 ISA 支持

  • IOMMU 的 MSI 重映射控制
    • 问题：确保设备只能触发自身中断。
    • 状态：进行中，高优先级。

4. 已完成的改进

  • 早期启动日志优化、窃取时间统计、SSE 支持、CoVE 规范、PMU 快照、原生调试、Zawrs 指令、嵌套虚拟化加速、QoS 支持、中断控制器虚拟化、IOMMU 设备直通等均已实现。

关键结论

RISC-V 虚拟化仍在中等至高级别优先级领域（如 PV-spinlocks、异步页错误、IOMMU 安全）存在差距，但部分核心问题（如 CoVE、PMU）已推进完成。

Slide 2

Survey of RISC-V virtualization gaps
  • Major architectures (x86_64, ARM64, etc) have virtualization support in various parts of the overall system
  • Based on above, we have following high-level gap areas:
    • SBI (or Paravirt) support for virtualization
    • ISA support for better virtualization performance
    • Non-ISA support for virtualization

Slide 3

SBI (or Paravirt) support for virtualization
  • PV-spinlocks
    • Hypervisor agnostic way for a Guest/VM to avoid lock holder preemption
    • TGs/SIGs: Hypervisors SIG, PRS TG
    • Proposal: None
    • PoC: https://lists.riscv.org/g/tech-prs/message/1211
    • Notes: Linux PoC - https://lore.kernel.org/all/20241227011011.2331381-1-guoren@kernel.org/
  • Timescaling (well-defined problem across architectures)
    • Hypervisor agnostic way for reporting timer frequency changes to Guest/VM at runtime
    • TGs/SIGs: Hypervisors SIG, PRS TG
    • Proposal: SBI TSCL extension
    • Notes: No activity in quite some time

Slide 4

SBI (or Paravirt) support for virtualization
  • Async page faults (well-defined problem across architectures)
    • Hypervisor agnostic way for reporting swapped-out G-stage pages to Guest/VM
    • Dependency: para-virt software injected synchronous faults using SBI SSE (supervisor software events)
    • TGs/SIGs: Hypervisors SIG, PRS TG
    • Proposal: None
  • Scalable confidential compute architecture
    • Hypervisor agnostic way to support scalable confidential compute
    • TGs/SIGs: Hypervisors SIG, AP-TEE TG, PRS TG
    • Proposal: CoVE specification

Slide 5

ISA support for better virtualization performance
  • Accelerating G-stage dirty page tracking
    • RISC-V ISA support for faster dirty page tracking in G-stage
    • TGs/SIGs: Hypervisor SIG, Priv IC, VirtMem TG
    • Proposal: None

Slide 6

Non-ISA support for virtualization
  • IOMMU control of MSI data when remapping MSIs
    • Ensure devices can only raise their own interrupts
    • TGs/SIGs: Hypervisor SIG, IOMMU TG
    • Proposal: None
