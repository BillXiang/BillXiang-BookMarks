Git 如何在 GitHub Action 工作流中获取push或merge之前的上一个提交

在本文中，我们将介绍如何在GitHub Action工作流中获取push或merge之前的上一个提交。

Git是一个分布式版本控制系统，通过跟踪文件的改动记录，可以帮助开发人员协同工作和管理代码。GitHub Action是GitHub提供的一种自动化工作流工具，可以根据触发条件执行预定义的任务。

当我们在GitHub Action工作流中执行任务时，有时需要获取push或merge之前的上一个提交，以进行一些特定的操作或分析。下面将介绍几种方法来实现这个目标。

使用git log

使用git log命令可以获取提交的历史记录。可以结合一些参数来筛选出上一个提交的相关信息。下面是一个示例：

name: Workflow Example

on:
  push:
    branches:
      - main

jobs:
  workflow_job:
    runs-on: ubuntu-latest
    steps:
    - name: Check out code
      uses: actions/checkout@v2

    - name: Get previous commit
      run: |
        git log -n 2 --pretty=format:"%h" | tail -n 1 > previous_commit.txt
        cat previous_commit.txt
Bash

在上述示例中，我们定义了一个工作流，在push到main分支时触发执行。在执行步骤时，我们先通过actions/checkout@v2来检出代码。然后使用git log -n 2 --pretty=format:"%h"命令来获取前两个提交的短SHA值，并通过tail -n 1来筛选出上一个提交的短SHA值，并将其保存到previous_commit.txt文件中。

使用GitHub APIs

GitHub提供了一系列的APIs来获取提交的信息。我们可以使用这些APIs来获取上一个提交的信息。下面是一个示例：

name: Workflow Example

on:
  push:
    branches:
      - main

jobs:
  workflow_job:
    runs-on: ubuntu-latest
    steps:
    - name: Check out code
      uses: actions/checkout@v2

    - name: Get previous commit
      run: |
        export PREVIOUS_COMMIT=(curl -s https://api.github.com/repos/GITHUB_REPOSITORY/commits?per_page=2 | jq -r '.[1].sha')
        echo $PREVIOUS_COMMIT > previous_commit.txt
        cat previous_commit.txt
Bash

在上述示例中，我们通过使用GitHub的APIs来获取前两个提交的信息，并通过jq来解析JSON响应，从中提取出上一个提交的SHA值。然后将该值保存到previous_commit.txt文件中。

使用GitHub Action的github-script操作

GitHub Action提供了一个github-script操作，可以使用JavaScript或TypeScript编写自定义的脚本。我们可以使用github-script操作来获取上一个提交的SHA值。下面是一个示例：

name: Workflow Example

on:
  push:
    branches:
      - main

jobs:
  workflow_job:
    runs-on: ubuntu-latest
    steps:
    - name: Check out code
      uses: actions/checkout@v2

    - name: Get previous commit
      uses: actions/github-script@v4
      with:
        script: |
          const commits = await github.repos.listCommits({
            owner: context.repo.owner,
            repo: context.repo.repo,
            per_page: 2
          });
          const previousCommit = commits.data[1].sha;
          console.log(previousCommit);
Bash

在上述示例中，我们使用了actions/github-script@v4操作来执行自定义的JavaScript脚本。在脚本中，我们通过调用github.repos.listCommits方法来获取前两个提交的信息，然后提取出上一个提交的SHA值并输出。

这些方法都可以用来获取push或merge之前的上一个提交。你可以根据自己的需求选择其中一种方法来实现这个目标。

总结

在本文中，我们介绍了如何在GitHub Action工作流中获取push或merge之前的上一个提交。通过使用git log命令、GitHub APIs或GitHub Action的github-script操作，我们可以轻松地获取上一个提交的SHA值。这些方法可以帮助我们在工作流中进行更加灵活和精确的操作和分析。希望本文对你有帮助！
