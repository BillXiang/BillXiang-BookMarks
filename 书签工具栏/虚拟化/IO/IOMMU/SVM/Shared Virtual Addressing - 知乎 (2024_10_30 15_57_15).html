  • 简介

共享虚拟寻址 (SVA)，也称为共享虚拟内存 (SVM)，允许与设备共享 CPU 地址空间，并简化设备驱动程序和用户空间进程的 I/O 内存管理。

  • SVA机制

未使用SVA， Device设备侧使用的是 IOVA, CPU侧使用的是VA

使用SVA的情况，IOVA == VA。


  • SVA的现状

1 Status of Arm SMMUv3 support for SVA - June 2021

There are a number of dependencies for SVA support in the SMMU driver. The following diagram describes all the required components, each of them a series of 2-10 patches.

|        [Fault API]      [IOMMU SVA] <----------- [IOASID]

1.1 Done

SMMU ATS

  • Support PCI Address Translation Service.
  • Merged in v5.2
  • git branch sva/current contains devicetree support.

Fault API

  • Generic device fault reporting for the IOMMU.
  • Merged in v5.3

IOASID

  • Add an IOASID allocator.
  • Merged in v5.5

SMMU PASID

  • Support PCI Process Address Space ID and SubstreamID.
  • Merged in v5.6 and v5.7.

IOMMU SVA

  • Generic management of process address spaces bound to devices.
  • The bind API has been merged in v5.3

SMMU SVA

  • Support sharing CPU address spaces with the SMMU.
  • arm64 part merged in v5.9
  • bind support merged in v5.11

I/O Page Faults

  • Handle device page requests using the core mm susbsystem.
  • merged in v5.13

SMMU Stall

  • Support SMMU Stall model for platform devices.
  • merged in v5.14

1.2 Todo

SMMU BTM

  • Support CPU-issued TLB invalidation
  • git branch sva/current

SMMU PRI

  • Support PCI Page Request Interface.
  • git branch sva/current
