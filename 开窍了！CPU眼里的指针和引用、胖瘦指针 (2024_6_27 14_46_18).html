指针和引用有什么区别？为什么有这些区别？cpu眼里的指针和引用是什么样的？

在看"Rust Language Cheat Sheet"的时候(强烈推荐！Rust语言知识备忘录，囊括几乎所有Rust知识点，拿去当字典用！)，猛然发现了"References & Pointers"的内容，这块内容一针见血的指出了引用和指针的区别：引用可以安全的访问第三方内存而指针则不安全。

图中引用和指针的内存布局是一样的，都有ptr和meta两部分内容，其中虚线框圈起来表示可选，图中的数字2/4/8表示位数(分别对应16位、32位、64位的操作系统)。

既然内存布局是一样的，那在cpu眼里执行的操作码和操作数理应一样，这一点我们可以用在线工具Compiler Explorer来验证。编写如下代码，并用Compiler Explorer在线工具编译得到汇编代码：

pub fn main() {

    let mut a = 123;

    test1(&a);

    test2(&a as *const i32);

    test3(&mut a as *mut i32);

}

#[no_mangle]

fn test1(input:&i32){}

#[no_mangle]

fn test2(input:*const i32){}

#[no_mangle]

fn test3(input:*mut i32){}

看到了吧，不管是引用、常指针、可变指针，在cpu眼里都一视同仁，它们都转换为rsp+4的栈地址。既然在cpu眼里都一样，为啥还要搞这么多花样呢？

指针的概念其实来自于汇编语言中的ptr，很直白，它表示一个内存地址。像汇编这种语言直接操作内存地址给程序员带来了巨大的灵活性，但也带来了潜在的危险，操作不当很容易出现内存段错误。

高级语言需要在语法上进行担保(guarantees)并由对应的编译器具体实现这些担保。

为了区别于指针，发明了"引用"的概念并用符号"&"表示。编译器在编译阶段可针对“引用”实施某些魔法，rust编译器提供了可变性检测、生命周期检测、作用域检测等骚操作，来为承诺的安全性担保。哪有那么多花好月圆，都是别人在为你负重前行！

胖瘦指针

很多指针和引用都可以携带元数据(metadata)，这些元数据可以是长度、或者是指向vtable的指针。拥有metadata的指针叫“胖指针”(fat pointer)，没有的则叫“瘦指针”(thin pointer)。

接下来我们用Compiler Explorer探查下，看在cpu眼里胖瘦指针长什么样。

下面是str在cpu眼里的样子：

pub fn main() {

    let age = 18;

    let name = "三角兽2024";

    test4(name);

}

#[no_mangle]

fn test4(input:&str){

   let len = input.len();

   let mut b = len+1;

   let addr = &input;

}

&str是带有metadata的“胖指针”，由指向数据的指针和切片长度组成。当name传入test4函数时，需要把name的这两部分数据都传递给函数，rdi寄存器保存了数据指针，esi保存了数据长度，可以看到是13，为什么是13呢？因为一个汉字占3个字节，3*3+4=13。

再来看看dyn特征对象动态分发的例子，特征对象也是一个胖指针，包含指向类型的指针和指向vtable的指针。下面是一个示例：

pub fn main() {

let a = A::new(18);

test5(&a);

a.f1();

}

trait T{

    fn sum(&self);

    fn f2(&self);

}

struct A{

    age:i32

}

impl T for A{

    fn sum(&self){

        1+1;

    }

    fn f2(&self){}

}

impl A{

    fn new(age:i32)->Self{

        A{age}

    }

    fn f1(&self){}

}

   

fn test5(args:&dyn T){args.sum();}

调用test5函数时，rdi寄存器存储了对象a的地址，rsi寄存器存储了vtable的地址。胖指针由于包含metadata，当发生函数调用时，需要多个寄存器来处理这些数据，这是胖指针的额外开销，这些额外的开销是值得的，它给程序带来了巨大的能量，发展出五花八门的“绝技”，借助它，rust实现了“动态分发”，与基于泛型的“静态分发”一起，实现了rust灵活的语言特性。
